# v1.3.0 Implementation Planning Finalization

Sean's questions cover git setup commands, Cloudflare DNS, environment strategy clarity, and config file creation.

This is documentation work only -- no production code exists yet. We're updating planning docs and creating 4 config files so implementation can begin with zero ambiguity.

- Order: (1) generate slug, (2) upload images using slug in paths, (3) create product with slug + image URLs

Add note: "These files must be created as actual files in the repo root. See Action Steps Pre-Flight."

**G. New sections to insert:**

- **Source of Truth Hierarchy** (GPT #9): Supabase = authoritative, Stripe = payment mirror, R2 = asset storage, Frontend = read-only
- **Stripe Sync Rules** (GPT #4): INSERT -> create Product+Price. Price change -> archive old, create new. Title/image change -> DO NOTHING
- **Webhook Contract** (GPT #3): Explicit step-by-step: verify sig -> check idempotency -> parse metadata -> upsert customer -> mark sold -> create orders -> record event.id
- **Cart Recovery Flow** (GPT #11): Full UX flow from 409 error through message copy, email capture, promo code display, and next actions
- **Agentic Pipeline Error Handling** (GPT #6): Retry image upload once then STOP. Never retry product creation blindly. Don't proceed if current step fails. Rollback: set available=false if images incomplete.
- **GA4 Event Definitions** (GPT #10): 11 events with parameters: page_view, view_product, add_to_cart, remove_from_cart, begin_checkout, purchase, newsletter_signup, contact_form_submit, search_filter, gallery_open, promo_code_generated

**H. API endpoint changes:**

- `api/products.ts` (GPT #1, #7, #12, #13):
  - Auth: `PRODUCT_API_KEY` instead of `SUPABASE_SERVICE_KEY`
  - Add field validation (required fields, price is integer, images >= 7)
  - Add slug generation + conflict check (409 on duplicate)
  - Add image role enforcement (1 hero, 1 thumbnail, 5+ gallery)
- `api/upload.ts` (GPT #8):
  - Auth: `PRODUCT_API_KEY` instead of `SUPABASE_SERVICE_KEY`
  - File type validation (JPEG, PNG, WebP, GIF, MP4, MOV)
  - File size limits (10MB images, 50MB videos)
- `api/webhook.ts` (GPT #2, #3):
  - Idempotency check at top of handler
  - Record event.id after processing
- `api/config.ts` (Sean Q9):
  - Add security note: serves ONLY public keys (pk_*, anon key, URL). No auth needed. Safe by design.
