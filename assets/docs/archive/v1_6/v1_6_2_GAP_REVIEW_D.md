## Part 1 — Folds confirmed (round-1 D-1…D-9 → v1.6.2), with the decisive evidence

- **D-1 (was round-1's headline would-render-wrong) — the warm hero spotlight now wins the cascade. CONFIRMED.** D7 REPLACES the in-page `.hero__spotlight` rule in place (`index.html:353–366`, the dark `multiply` vignette) with the warm `screen` glow. Decisive fact: a repo-wide search finds exactly **two** `hero__spotlight` references — the element (`index.html:166`) and that one in-page rule (`:353`); there is **no `.hero__spotlight` in `styles.css`**. So after the in-place replace there is a single `.hero__spotlight` rule, in the in-page `<style>` that loads after `styles.css` (`<link>` at `index.html:71`) — it wins unconditionally. The warm rule's `rgba(var(--glow-color, 212, 175, 122), 0.25)` carries its own warm fallback, so the color is correct even if D6's `:root` token weren't applied. Reduced-motion is folded into the in-page block (`:380–382` → expanded): `.hero__video { display:none }` + `.hero__media { background-image: poster }` + `.hero__spotlight { animation:none }`, and `.hero__media` is `position:absolute; inset:0` (`styles.css:948–952`) so the poster background fills the hero. The `@supports` counter-scroll (`:368–378`) is left intact and is transform-compatible with the `<img>`→`<video>` swap.

- **D-2 — the product `.product-aside` sticky mechanic is sound, and the a11y miss is fixed. CONFIRMED.** The aside gets `grid-area: aside` spanning the gallery+story rows (`"gallery aside" / "story aside" / "media ."`), with `align-self:start; position:sticky; top:calc(--header-height + --space-lg)`. A sticky grid item is constrained by its **grid area**, and `align-self:start` makes the aside content-height, so it pins through gallery+story and releases at the media row — exactly FEEDBACK #3. The card's own sticky is neutralized inside the aside by `.product-layout > .product-aside > .product-sticky-card { position:static }` — specificity (0,0,3,0) beats the base `.product-sticky-card { position:sticky }` (0,0,1,0) **and** is later in source order, a double-win. Crucially, the base card-sticky is scoped to `@media (min-width:768px)` (`styles.css:885–890`), so on mobile the card is already static — the taller aside does **not** make it stick on mobile (FEEDBACK #8 holds). After the moves the DOM order is `gallery → aside(card+details) → story → media`, so the BUY CTA is 2nd in reading/focus order (before the media controls) — the round-1 D-2b a11y miss is resolved, and DOM order equals the mobile visual order.

- **D-3 — desktop density is scoped, no global token survives, and it bites the right pages. CONFIRMED.** D5 now adds `@media (min-width:1024px){ .density-compact .section{…} .density-compact .section-loose{…} }` and a `density-compact` class on `cart.html` / `checkout.html` `<main>` — it does **not** touch `:root --space-2xl/3xl`, so `--container-padding` gutters and every other page's rhythm are untouched. Verified it actually applies: both pages wrap content in `<div class="container section" …>` (`cart.html:150`, `checkout.html:150`), so `.density-compact .section` (0,0,2,0) overrides the base `.section` (0,0,1,0). The other `<section>`s on cart (`:183`, `:219`) use inline padding (no `.section` class), so the scope can't over-tighten them.

- **D-4 — the inline `gap` is gone from both layouts and the breakpoint gap can win. ## Part 2 — Re-scan of the touched regions (non-monotonic). No blockers; ranked low → lowest

1. **LOW · T2.1a / T2.4 — the `gifloop` fixture has no `poster`, so its reduced-motion fallback shows the first video frame, not a designed still.** T2.4 describes the reduced-motion GIF-like fallback as "paused **poster** + controls," and the click-to-play fixture carries a `poster`, but `test-media-gifloop` has none. With `preload="metadata"` most browsers show the first frame as the poster — not broken, just not the "still" the wording implies. **Fix:** add a `poster` URL to the `gifloop` fixture (the existing `hero-bg-image-not-anim.jpg` works), or note in T2.4 that a poster-less GIF-like clip falls back to its first frame.

2. **LOW · T2.1 matrix — no single fixture exercises "MP4 first, YouTube after" together.** `gifloop` and `clicktoplay` are MP4-only; `test-media-youtube` is YouTube-only. The MP4→YouTube sort (`populateMedia:2429`, FEEDBACK #5) is therefore proven by code-reading, not by a rendered fixture. **Fix (optional):** give one fixture both an MP4 and a YouTube item so the on-page order is visually provable, or state that the ordering is code-verified and the single-type fixtures are sufficient.

3. **LOW · D2 — `.product-aside` has no internal gap, and `.product-details` reflows into the narrow right column.** The aside is a plain block; with the details' `margin-top` dropped, the details abuts the card at the details' own `border-top` (the intended divider, but it sits flush against the card with no breathing room). Separately, moving `.product-details` from a full-width section into the ~360–400px aside reflows its `<h2>Details</h2>` (40px) and six icon+text feature rows into a narrow column. Both are FEEDBACK #3-intended (details in the right rail) and the spec already flags the `border-top`/`padding-top` trim — but the narrow-column heading scale / row wrapping is worth an explicit eye-check on render. **Fix (render-tune):** trim the details `border-top`/`padding-top` (or add a small `.product-aside` gap) and eye-check the `<h2>` scale + feature-row wrapping at the column width.

4. **LOW (optional) · D4/D8 — the static shop fallback tiles still list non-miniature types after the Type filter drops them.** D4 (C3) removes the `storybook`/`printable` options (miniatures-only), but `shop.html`'s static fallback grid still hard-codes `data-product-type="printable"` (`:307`) and `story-lofts`/`seasonal` tiles. `shop.js` overwrites the grid on load (and `initFromURL` safely no-ops on a dropped value — `querySelectorAll().forEach` over an empty list throws nothing), so this is invisible in practice; only the never-shown fallback diverges. **Fix (optional consistency):** mirror miniatures-only in the static fallback tiles too, so the fallback can't contradict the live filter set.

5. **NOTE (not a gap) · D7 depends on D6 shipping in the same build.** The warm spotlight's `animation: glow-breathe 18s` uses D6's `@keyframes glow-breathe` and reads D6's `:root --glow-color`, both added to `styles.css`. Both are in this build; the `var(--glow-color, 212,175,122)` fallback de-risks the color, so the worst case if D6's keyframe were somehow absent is a static-but-warm spotlight (no breathing), never a broken render. Build D6's CSS with D7.
